# 产品需求文档（PRD）— 亚马逊业绩报告 Web

> **文档性质**：描述当前**已实现**能力、数据流、交互边界与已知限制，便于评审、交接与迭代。  
> **维护方式**：新功能请更新 **§8 功能清单**、**§12 backlog**、**§13 变更记录**；重大行为变更注明日期与负责人（可选）。  
> **与工程关系**：本文件**不参与**构建与运行（`npm run dev` / `npm run build` **不读取**本文件），**对 App 使用零影响**。

---

## 1. 产品概述

### 1.1 定位与价值

面向亚马逊运营 / 管理团队的 **浏览器内经营看板**：通过导入多类报表（Excel/CSV），在本地完成解析、清洗、聚合与对比，统一查看 **P&L、广告流量、FBA 库存**，并辅以 **多维下钻子表、退货/评论/搜索词专题、利润试算、AI 问答**。

### 1.2 目标用户

| 角色 | 典型诉求 |
|------|----------|
| 一线运营 | 按店铺/ASIN/负责人看达成、环比同比、费用结构；下钻找异常款。 |
| 组长/主管 | 多维度拆解、目标进度、广告与库存风险。 |
| 管理层 | 汇总卡片 + 子表复制到飞书汇报；可选 AI 摘要（需注意数据出境）。 |

### 1.3 技术形态（产品可读版）

| 项 | 说明 |
|----|------|
| 形态 | 单页 Web 应用（React + TypeScript + Vite），纯前端运行。 |
| 数据 | **不上传自有服务器**；解析与计算在用户浏览器内完成。 |
| 持久化 | **IndexedDB** 缓存业绩/目标/库存/退货/评论及筛选元数据；刷新后可恢复。 |
| 大模型 | 可选 **Gemini**、**通义（DashScope）**、**智谱** 等；密钥与提示词多存于 **localStorage**（安全见 §11）。 |

## 2. 信息架构与主界面

### 2.1 布局

- **左侧**：固定 **筛选侧栏**（时间、国家、店铺、品牌、负责人、类目、父/子 ASIN、品名等 + 数据/目标告警）。
- **右侧**：顶栏（**月度/周度**切换、世界时钟、**导入/更新数据源**）+ 可滚动主内容区。
- **全局**：右下角 **AI 助手（ChatBot）** 浮层（有数据且配置 API 时可用）。

### 2.2 主内容区模块顺序（有有效 `processedData` 时）

1. **DashboardOverview**：指标卡片（月度为目标达成进度条；周度为当周绝对值卡片）。
2. **PLTable（P&L）**：损益与结构指标 + 环比/同比/目标（与表格渲染器一致）。
3. **TrafficTable（流量与广告）**：广告花费/销售、分类型 SP/SD/SB/SBV、展示点击、CVR、ACoS 等。
4. **InventoryTable（库存）**：基于当前筛选的 FBA 聚合与库龄等（可下钻库存子表）。

无数据时展示空状态，引导导入 Excel。

### 2.3 月度 vs 周度

| 模式 | 数据源 | 默认时间行为（概要） |
|------|--------|----------------------|
| **月度** | `performanceData`（月度业绩表解析结果） | 筛选为自然日区间；对比 **上期**、**去年同期**；目标按月与筛选维度匹配。 |
| **周度** | `weeklyData`（周度业绩表） | 开启周度时切换到周数据；侧栏/工具函数使用 **ISO 周** 等逻辑（详见 `utils.ts`）。 |

**导入行为**：每次完整导入成功后 **默认回到月度模式**（避免已开周度但未上传周表导致整页无数据）。

## 3. 数据接入

### 3.1 上传入口与流程

- 入口：**「导入数据源」/「更新数据源」** → `DataUploadModal`。
- **必填**：**月度业绩表**（`monthly`）；否则无法提交。
- **可选**：周度业绩、全年目标、FBA 库存、退货报告、评论报告（见下表）。
- 支持 **按槽位单文件** 与 **批量选择多文件**；批量时按 **文件名规则** 自动落入槽位（可并存未识别文件列表）。
- 导入完成后先展示 **解析报告/缺表提示**，用户确认 **「开始分析」** 后再关闭弹窗（与主表状态同步）。

### 3.2 槽位与文件名识别（`uploadFileClassifier.ts`）

| 槽位键 | 展示名 | 必填 | 识别要点（简述） |
|--------|--------|------|------------------|
| `monthly` | 月度业绩表 | **是** | 月度/月报/monthly、泛化业绩 performance/sales 等；与周度规则互斥优先级见代码顺序。 |
| `weekly` | 周度业绩表 | 否 | 周度/weekly/年周/W+周序 等。 |
| `target` | 全年目标表 | 否 | 目标/target/kpi/goal 等。 |
| `inventory` | FBA 库存表 | 否 | 库龄/aged/inventory report、FBA 库存特征词等；**退货关键词优先于** 裸 FBA。 |
| `refund` | 退货报告 | 否 | 退货/退款/return report/removal 等大量别名。 |
| `review` | 评论报告 | 否 | 评论/review/VOC（排除 preview）等。 |

未识别文件进入 `unrecognized` 列表提示用户；可选槽位未传时 `listMissingOptionalSlots` 给出 **缺表提醒**（不阻断月度导入）。

### 3.3 解析与字段映射

- 核心逻辑：`dataLoader.ts`（SheetJS / `xlsx` 读表、列别名、类型转换、周/月解析等）。
- **列显示名与别名**：`COLUMN_DISPLAY_NAMES`、`PERFORMANCE_ALIASES`、库存等专用别名（详见代码）；兼容中英文及运营常见表头。
- 导入结果可带 **`DataSourceDebugInfo`**（debug），上传弹窗用于展示匹配列、警告等（具体字段以类型定义为准）。

### 3.4 搜索词（关键词）数据 — **独立入口**

- **不在** 主上传六槽位内。
- 用户在 **流量表** 相关入口打开 **关键词分析弹窗**（`KeywordAnalysisModal`），在弹窗内 **单独上传** 搜索词报表并由 `parseSearchTermReport` 解析。
- 全量重新导入主数据时，`App` 会 **清空内存中的搜索词数据**（`setSearchTermData([])`）；已导入的搜索词文件需按需重新在弹窗内上传。

## 4. 核心数据对象（业务字段概要）

以下为 **逻辑模型** 摘要，完整字段见 `types.ts`。

### 4.1 业绩行 `DataRow`

- **维度**：日期 `YYYY-MM-DD`、国家、店铺、品牌、负责人、二级分类、父/子 ASIN、品名等。
- **经营指标**：销量、销售额、毛利、FBA 可售、Sessions、各项成本（头程/采购/仓储/FBA 费/退款/佣金）、广告花费与销售（总及 SP/SD/SB/SBV）、展示/点击、自然订单/点击、评分与评论数等。

### 4.2 目标行 `TargetRow`

- 维度与业绩对齐（品牌/国家/店铺/负责人/类目/ASIN/品名等）+ **月份 `YYYY-MM`**。
- 指标：销量/销售额/毛利额/毛利率/广告花费等目标字段（详见类型）。

### 4.3 库存行 `InventoryRow`

- 含 MSKU/FNSKU/SKU、多级类目、品牌/负责人/国家、FBA 各状态数量与成本、**库龄分段**（0–30 天 … 365+ 天）、AWD 等（列位与导入映射以 `dataLoader` 为准）。

### 4.4 退货行 `RefundRow`

- 订单、SKU/ASIN、品名/长标题、国家、原因、处置方式、买家备注、数量、金额、退货日期等。

### 4.5 评论行 `ReviewRow`

- 日期、ASIN/父 ASIN、星级、标题、正文、映射品名/国家、点赞、评论链接等。

### 4.6 搜索词行 `SearchTermRow`

- 投放词、匹配方式、用户搜索词、曝光/点击/CPC/花费、广告订单/销售额、活动/广告组/组合名等。

## 5. 计算逻辑（产品需知）

### 5.1 时间区间与对比期

- `calculatePeriodDates` 根据当前筛选的起止日期，推导 **当期 / 上期（环比）/ 去年同期（同比）** 三段日期（具体日历规则以 `utils.ts` 为准）。
- `filterData` 按筛选器与时间窗过滤 `DataRow`；`aggregateData` 聚合成 `AggregatedData`。

### 5.2 缺失对比期数据

- 若上期或去年同期无行，聚合结果为 **`null`**，表格中对应环比/同比行 **隐藏或显示占位**（避免与「真实为 0」混淆）。产品文档需与业务确认期望展示。

### 5.3 目标与 pacing

- `getTargetForSelection` 结合目标表与筛选、时间窗匹配目标；`getPacingRatio` 用于按自然日进度折算（**当前实现可能存在占位/简化**，以代码注释及业务校准为准）。

### 5.4 数据与目标完整性告警

- `analyzeDataCompleteness`、`analyzeTargetCompleteness` 等生成 **warnings**，在侧栏等处展示（文案以界面为准）。

## 6. 筛选侧栏（`SidebarFilters`）

### 6.1 行为特点

- **Excel 式逻辑**：多选下拉；**空选表示全选**（未排除任何项）；特殊 token 表示「全不选」等（见 `EnhancedDropdown`）。
- 支持选项内 **搜索**、全选可见、清空等。
- **时间**：月度为日期范围；周度与 ISO 周/业务周控件联动（组件内含周选择、历史范围等）。
- **告警展示**：与 `processedData.warnings`、目标完整性等绑定。

### 6.2 用户指南

- 侧栏提供 **平台使用指南** 弹窗（步骤：接入数据 → 筛选 → 看板 → 下钻 → AI）。

## 7. 主表模块（概要）

### 7.1 DashboardOverview

- **月度**：五张 **进度型卡片** — 销量/销售额/毛利额/毛利率 **达成**（对比目标）、**广告预算消耗**（反向配色：超支警示）。
- **周度**：五张 **绝对值卡片** — 本周销量、销售额、毛利额、毛利率、广告花费（无目标进度条版本）。

### 7.2 PLTable / TrafficTable / InventoryTable

- 由 `TableRenderers.tsx`、`InventoryTable.tsx` 等渲染；含 **转置/对比行**、点击打开 **详情子表**、跳转 **计算器/退货/评论/关键词** 等入口（以当前 UI 为准）。
- **库存表** 可打开 **库存维度子表**（`DetailAnalysisModal` `type=Inventory`）。

## 8. 弹窗与专项功能（详细）

### 8.1 详情分析子表 `DetailAnalysisModal`

| 能力 | 说明 |
|------|------|
| 类型 | `PL`（业绩）、`Traffic`（流量广告）、`Inventory`（库存）。 |
| 维度拆解 | 多选维度：国家、年月/年周、二级分类、品牌、店铺、负责人、父/子 ASIN、品名等（库存模式下去掉不适用项）。 |
| 列配置 | P&L / 流量 / 库存三套列；含计算器入口（毛利率）、评论入口（评分列）、退货入口（退款占比）、趋势入口等。 |
| 列头筛选 | 每维度列漏斗筛选，过滤 **逻辑行**（与导出/AI 范围一致）。 |
| 排序 | 点击表头排序；支持维度列与指标列。 |
| 虚拟滚动 | 先渲染部分行，滚动加载更多（性能优化）。 |
| 总计行 | 底部 **总计 (Total)**，与当前筛选范围内数据聚合一致。 |
| 趋势图 | `TrendChartModal`：按当前子表范围或单行维度查看指标趋势（ raw 月/周数据来自 App 传入）。 |
| **复制表格（飞书）** | 写入剪贴板 **HTML + 纯文本 TSV**：含表头灰底、总计行、涨跌颜色、目标达成颜色、**居中**、**销量/销售额/毛利额加宽（约 1.4×）**、`table-layout`/`colgroup`/列宽属性等（飞书若弱化样式属编辑器限制）。 |
| 子表 AI | 基于 **当前筛选+排序后的行** 生成 Markdown 报告；**独立提示词配置键**（P&L / 流量各一）；依赖已配置 API。 |

### 8.2 退货分析 `RefundAnalysisModal`

- 数据源：`refundData`（导入退货表）+ `rawPerformance`（用于关联或上下文）。
- 支持 **按主界面筛选或子表传入的 override 筛选** 打开。
- 内含 **AI 生成报告、规则配置、数据变更回写** 等（以界面为准）；可 **导出 Markdown** 等能力见组件实现。

### 8.3 评论分析 `ReviewAnalysisModal`

- 数据源：`reviewData`；`initialFilters` 来自主筛选。
- **AI 舆情分析**、PDF/图片导出（html2canvas + jsPDF）等（以界面为准）。

### 8.4 关键词（搜索词）分析 `KeywordAnalysisModal`

- 弹窗内 **单独上传** 搜索词报表；解析后表格化展示。
- **曝光分层**（高/中/低）、**效率四象限**（点击×转化）、语义分类、排序、**AI 分析**、**提示词设置**、阈值与 CTR/CVR 基准配置等。

### 8.5 利润试算 `ProfitSimulatorModal`

- 基于 `rawData`（月度业绩）与当前 `filters`，支持从子表带 **品名/国家** 初值；用于 **利润模型试算**（具体字段与公式以组件为准）。

## 9. 全局 AI 助手 `ChatBot`

| 项 | 说明 |
|----|------|
| 输入 | 用户自然语言问题。 |
| 上下文 | 当前筛选下的 **多维聚合**（负责人/国家/品牌/父 ASIN 等）、**Top 单品**摘要、库存结构等（构建逻辑见 `ChatBot.tsx` / `getDeepContext`）。 |
| 输出 | 流式或完整文本回复（依接入的 API）。 |
| 配置 | `ChatApiSettingsPanel` / `aiApiConfig`：提供商、密钥、模型、代理 URL；**localStorage** 持久化。 |
| 提示词 | `PromptSettingsModal` 可维护 ChatBot 系统提示与模板（与 **子表 AI** 配置键分离）。 |

**合规**：上下文含真实经营数据，上传至第三方模型前需业务/法务确认。

## 10. 本地持久化（`db.ts`）

| Store 名 | 内容 |
|----------|------|
| `monthly` | 月度业绩 `DataRow[]` |
| `weekly` | 周度业绩 `DataRow[]` |
| `targets` | 目标 `TargetRow[]`（可能经 `enrichTargetData`） |
| `inventory` | 库存 `InventoryRow[]` |
| `refunds` | 退货 `RefundRow[]` |
| `reviews` | 评论 `ReviewRow[]` |
| `meta` | 当前 `FilterState`（含起止日期等） |

**启动流程**：并行 `loadFromDB`；若有 `meta` 则恢复筛选；否则若有月度数据则默认 **最近月** 为时间区间。

**全量导入**：`clearDB` 后重写各 store；**搜索词** 仅内存，不写入上述 DB（见 §3.4）。

## 11. 安全、隐私与运维注意

| 风险 | 说明 |
|------|------|
| API Key | 存于浏览器（localStorage / 构建注入），**存在泄露与盗用风险**；生产环境建议后端代理或企业密钥方案。 |
| 数据出境 | AI 请求将聚合/明细摘要发往境外或第三方；需合规评估。 |
| 本地缓存 | IndexedDB 数据留在本机；共用电脑需注意清缓存或隐私模式。 |
| 粘贴飞书 | HTML 表格依赖对方编辑器解析，极端情况样式/列宽与预期不一致。 |
